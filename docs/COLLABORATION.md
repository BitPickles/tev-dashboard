# 协作指南

## 角色分工

| 角色 | 负责人 | 职责 |
|-----|-------|-----|
| 数据调研 | AibotClawd (AI) | 搜索、抓取、整理协议数据 |
| 数据校验 | 用户 | 纠错、补充、提供资料 |
| 网站维护 | AibotClawd (AI) | 更新数据、修复问题 |
| 产品方向 | 用户 | 决定覆盖哪些协议、功能优先级 |

## 工作流程

### 新增协议
1. 用户告诉我要添加哪个协议
2. 我进行 LLM 调研，输出初版数据
3. 用户审核，指出问题
4. 我修正后入库

### 数据更新
1. 定期（季度/年度）触发调研任务
2. 或用户主动告知新年报发布
3. 我更新数据，用户确认

### 纠错
用户可以直接说：
- "Pendle 的 TEV 应该是 $35M，不是 $31M"
- "Aave 出了官方年报，链接是 xxx"
- "这个估算逻辑不对，应该这样算..."

我会立即修正并说明改动。

## 沟通格式

### 用户提供资料
```
协议：Pendle
来源：https://xxx
内容摘要：2025年净收入 $40M，其中...
```

### 我的调研输出
```
协议：Lido
置信度：medium
数据：
  - Gross Fees: $xxx (来源: DefiLlama)
  - Net Revenue: $xxx (估算，因为...)
  - TEV: $xxx (链上追踪)
待确认：
  - stETH 分润比例是否变化？
```

## 数据质量标准

| 置信度 | 标准 | 展示方式 |
|-------|-----|---------|
| ✅ high | 官方年报/公告 | 正常显示 |
| ⚠️ medium | 第三方数据 + 部分估算 | 标注"估算值" |
| ❓ low | 主要靠推断 | 标注"待确认" + 不纳入排行 |

## 争议处理

如果我和用户对数据有分歧：
1. 我说明我的数据来源和推理
2. 用户说明他的依据
3. 以更可靠的来源为准
4. 如果都不确定，标注为"待确认"
