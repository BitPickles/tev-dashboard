## 部署经验总结（写入长期记忆）

### GitHub Pages + Next.js 静态部署三件套
1. `touch out/.nojekyll` — 否则 `_next/` 目录被 Jekyll 过滤
2. `basePath: "/repo-name"` — 子路径部署必须配
3. 清 `.next` 缓存再 build — 否则 basePath 不生效

### Vercel 部署踩坑
- 带 `.git` 目录部署会读 Git author，Hobby plan 不支持协作者 → 用无 `.git` 副本
- `.env` 文件会覆盖 Vercel Dashboard 环境变量 → 部署副本不带 `.env`
- Prisma prebuilt 部署需要加 `binaryTargets: ["native", "linux-arm64-openssl-3.0.x"]`
- Supabase 新加坡 + Vercel 美东 = 跨洋连接池超时，无法解决
