## 二、矿田（首页 `/`）

### 2.1 机会卡片

每个机会占一行，卡片展示以下信息：

**必显字段：**

| 字段 | 说明 |
|------|------|
| 平台 + Logo | Binance / OKX / Aave 等 |
| 机会名称 | 简短标题 |
| APR / APY | 醒目大字展示 |
| 收益构成 | 拆分：基础收益 + 奖励/空投部分 |
| 容量 | 已用/总量 + 进度条百分比 |
| 开始时间 | 机会起始时间 |
| 结束时间 | 机会截止时间 |
| 剩余时间 | 实时倒计时（X天X小时X分） |
| 标签 | 币种、链、类型（Staking/Launchpool/Earn...） |

**可选辅助信息：**
- 锁仓期（灵活/7天/30天...）
- 最低参与金额
- Boss 评级（🟢推荐 / 🟡观望 / 🔴谨慎）
- 角标："新" / "即将结束" / "热门" / "即将开始" / "即将满额"

### 2.2 筛选栏

**默认状态：** 只显示 status=active（进行中）的机会

| 筛选器 | 类型 | 选项 |
|--------|------|------|
| 渠道 | 多选 | CEX / DeFi |
| 币种 | 多选 | USDT / USDC / BTC / ETH / BNB / 其他 |
| 容量 | 范围 | 不限 / <$100K / $100K-$1M / >$1M |
| 持续时间 | 范围 | <7天 / 7-30天 / 30-90天 / >90天 / 灵活 |
| APR 排序 | 单选 | 从高到低 / 从低到高 |

**不做的：**
- ❌ 不要平台筛选
- ❌ 不要 APR 区间筛选

**默认排序：** 由运营在后台手动设置（sortOrder 字段），不固定规则。

### 2.3 查看全部机会

页面底部固定按钮：`查看全部机会 →`

跳转到 `/all` 页面：
- 同样的卡片布局和筛选功能
- 默认显示所有状态（进行中 + 已结束 + 即将开始）
- 已结束的机会灰显，显示实际收益
- 新增状态筛选：进行中 / 已结束 / 即将开始

## 三、机会详情页（`/opportunity/[id]`）

点击卡片进入。

### 3.1 顶部概览
- 机会名称 + 平台 + 状态标签
- APR 大字 + 收益构成拆分
- 倒计时 + 容量进度条
- **一键参与按钮**（跳转到交易所/协议对应页面）

### 3.2 参与指南
- **怎么参与** — 分步骤说明，尽量清晰到截图级
- **需要什么** — 所需币种、最低金额、前置条件
- **收益怎么算** — 计算逻辑、复利/单利、发放频率、发放代币
- **怎么退出** — 赎回规则、延迟时间、滑点

### 3.3 Boss 评价
- **一句话结论**（thesis）
- **适合谁** — 稳健型 / 激进型 / 大资金 / 小资金
- **现在该做什么** — 立即参与 / 观望 / 等回调再进
- **风险提示** — 合约风险 / 币价风险 / 平台风险
- **退出信号** — 什么条件下该撤

### 3.4 动态更新流
- 按时间倒序的更新记录（类似推文流）
- 内容：收益率变化、容量变化、规则调整、Boss 补充评论
- 让机会成为"活的"，不是静态页面

### 3.5 侧边/底部信息
- 相关链接（官方公告、合约地址、审计报告）
- 相关机会推荐
- 分享按钮
- 收益计算器（输入金额，估算到期收益）

## 四、登录与权限

### 4.1 浏览权限
- **无需登录即可浏览**所有机会列表和详情
- 钱包连接仅在收藏、个性化等功能时要求

### 4.2 签名登录
- 用户连接钱包后签名一次
- Session 有效期：**30 天**（从当前 7 天延长）
- 到期后才要求重新签名
- 中间不打扰用户

### 4.3 测试期
- `ALLOW_ANY_SIGNED_USER=true`：签名即可访问，不卡白名单
- 正式上线后可切换为白名单模式

## 八、设计原则

1. **移动端优先** — 卡片布局天然适合手机，确保触控体验
2. **信息密度高** — 一眼看到关键数据，不要大留白
3. **暗色主题** — 延续 Crypto3D 设计系统
4. **实时感** — 倒计时、容量进度条要有动态感
